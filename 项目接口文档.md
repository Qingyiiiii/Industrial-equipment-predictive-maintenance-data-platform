# 项目接口文档

## 1. 项目边界

项目名称：MetroPT-3 工业设备预测性维护数据平台。

业务域：`metropt_quality`。

项目定位：

- 大数据平台学习项目。
- 工业设备预测性维护作品集项目。
- 支持离线链路、实时链路、扩展查询、运维巡检和最终交付包。

当前阶段结论：

- P0-P8 已完成。
- P9-P14 已形成扩展阶段复验与交付自动化证据。
- 完善阶段已补 2026-06-09 当前最新 standard P14 正式复验证据和交付记录。
- 当前不再以修底座或继续扩展为主；实时链路、Trino/Doris、P10 重跑耗时优化已重新纳入可选优化指南，但不改变 P14 standard 边界。

## 3. 数据接口

### 3.1 原始数据

| 数据 | 路径 |
| --- | --- |
| 本地 CSV | `<repo-root>\datas\MetroPT3_AirCompressor.csv` |
| 本地数据说明 PDF | `<repo-root>\datas\Data Description_Metro.pdf` |
| 本地数据说明 TXT | `<repo-root>\datas\Data Description_Metro.txt` |
| HDFS CSV | `hdfs:///lakehouse/projects/metropt_quality/raw/MetroPT3_AirCompressor.csv` |
| HDFS 数据说明 PDF | `hdfs:///lakehouse/projects/metropt_quality/raw/Data_Description_Metro.pdf` |

注意：

- 项目当前验收基线中的 ODS 行数是 `1,516,948`。
- 数据说明中提到 `15,169,480 data points`，这是多变量时间序列的数据点概念，后续不要和“CSV 行数”混用。

### 3.2 数据集语义

数据来自列车空气压缩机 Air Production Unit。采集频率为 1Hz，时间范围为 2020 年 2 月到 2020 年 8 月。

数据说明明确指出该数据集可用于：

- failure prediction
- anomaly detection
- remaining useful life prediction
- classification
- regression
- deep learning
- machine learning

传感器构成：

- 7 个模拟信号：压力、油温、电机电流等。
- 8 个数字信号：阀门、电气开关、油位、流量脉冲等。

当前项目使用的故障窗口：

| 编号 | 开始时间 | 结束时间 | 故障 | 严重程度 |
| --- | --- | --- | --- | --- |
| 1 | `2020-04-18 00:00:00` | `2020-04-18 23:59:59` | Air leak | High stress |
| 2 | `2020-05-29 23:30:00` | `2020-05-30 06:00:00` | Air leak | High stress |
| 3 | `2020-06-05 10:00:00` | `2020-06-07 14:30:00` | Air leak | High stress |
| 4 | `2020-07-15 14:30:00` | `2020-07-15 19:00:00` | Air leak | High stress |

所有 P2+ 脚本化验收运行结果默认写入：

```text

```

P14 标准 cluster 复验入口：

```bash
cd /home/common/tmp/pycharm_Design
bin/p10_p9_master_validation.sh \
  --mode standard \
  --allow-swapoff \
  --realtime-max-events 1000 \
  --realtime-wait-seconds 60 \
  --query-timeout 300
```

P14 三档复验模式：

| 模式 | 命令 | 说明 |
| --- | --- | --- |
| smoke | `bin/p10_p9_master_validation.sh --mode smoke` | 日常开工检查；只覆盖依赖、语法、P7、基础服务和 Hive 小样例；不能作为正式验收 PASS |
| standard | `bin/p10_p9_master_validation.sh --mode standard --allow-swapoff --realtime-max-events 1000 --realtime-wait-seconds 60 --query-timeout 300` | 正式 P14 复验；不得使用 `--skip-*` 作为完整 PASS |
| full | `bin/p10_p9_master_validation.sh --mode full --allow-swapoff` | 夜间或深度复验；replay 和 query timeout 更大，允许长跑 |

P14 标准验收输出：

| 输出 | 说明 |
| --- | --- |
| `/home/common/tmp/pycharm_Design/data/metropt_quality/validation_runs/p14_master_validation_20260609_020821/` | 当前最新正式 standard P14 run_dir，`final_status: PASS`，`SUMMARY pass=18 warn=0 skip=0 fail=0` |
| `/home/common/tmp/pycharm_Design/data/metropt_quality/validation_runs/p14_master_validation_20260607_054200/` | 已通过的标准 run_dir，`SUMMARY pass=18 warn=0 skip=0 fail=0` |
| `/home/common/tmp/pycharm_Design/data/metropt_quality/validation_runs/p14_master_validation_20260608_050123/` | 历史复验 run_dir，`final_status: PASS_WITH_WARNINGS`；业务步骤全部 PASS，唯一 WARN 为 P7 资源余量低 |
| `validation_report.md` | run_dir 内自动生成的完整复验报告 |
| `summary.tsv` | run_dir 内标准 PASS/WARN/SKIP/FAIL 汇总 |
| `p14_steps.tsv` | 每一步 return_code、日志路径、child run_dir、next_action |
| `p14_hive_results.tsv` | Hive P9 dashboard SQL 返回码、返回行数、SQL 文件和日志路径 |
| `data/metropt_quality/analysis/reports/p14_master_validation_report_20260607_054200.md` | 本机归档的正式报告副本 |

P14 失败定位规则：

- 先看 `p14_steps.tsv` 中第一条 `FAIL` 或 `WARN`。
- 再打开该行的 `log` 和 `child_run_dir`。
- 按该行 `next_action` 执行修复或单步复跑。
- 不允许手工把 `FAIL` 改成 `PASS`；必须保留重新运行后的证据路径。

## 9. 开发规范

- 新增脚本统一放入 `bin/`。
- 新增分析代码优先放入 `analysis/`。
- 新增 Spark 离线生产链路放入 `src/`。
- 新增实时链路放入 `streaming/`。
- 新增配置优先扩展 `config/metropt_quality.*.yaml`。
- 新增长期文档使用 Markdown，不使用临时 `.txt` 作为正式结论。
- 新增验收必须输出到 `validation_runs`，并保留 `run_id`、命令、返回码和关键日志路径。
